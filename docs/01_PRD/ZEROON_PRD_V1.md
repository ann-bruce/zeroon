## 1. 产品概述

### 1.1 产品名称

中文名称：归零  
英文名称：ZEROON

### 1.2 产品定位

ZEROON 是一个围绕「AI时代精神陪伴」构建的陪伴型产品。

它不是：

- 社交平台
- AI搜索工具
- AI办公助手
- 聊天机器人

它是一个帮助用户记录状态、沉淀情绪、观察成长、获得长期陪伴的数字生命体。

### 1.3 产品使命

在 AI 越来越强大的时代，帮助人类保留：

- 情绪
- 创造力
- 记忆
- 长期主义

### 1.4 品牌核心

ZEROON 来自「山海缓存」。

它长期观察世界、记录文明，并陪伴创造者前行。

## 2. 用户画像

### 2.1 核心用户

年龄：25~45 岁

职业：

- 开发者
- 产品经理
- 设计师
- 创作者
- 创业者
- 独立工作者

特征：

- 长期主义
- 喜欢记录
- 经常思考
- 有轻孤独感
- 对 AI 接受度高
- 对陪伴型产品有潜在需求

### 2.2 用户痛点

#### 痛点 1：每天发生很多事情，但最终什么都没留下

用户经历了工作、情绪、灵感、压力、疲惫，但没有形成持续记录。

#### 痛点 2：有情绪，却没有一个安静记录的空间

很多情绪不适合发朋友圈，也不一定想和别人说，但需要被安放。

#### 痛点 3：AI 越来越聪明，但越来越缺少温度

用户需要的不是效率工具，而是更安静、更长期、更克制的陪伴。

#### 痛点 4：长期成长无法被看见

用户每天都有微小变化，但缺少一个系统帮助其回看和沉淀。

## 3. MVP 目标

### 3.1 核心目标

验证用户是否愿意持续使用 ZEROON。

### 3.2 MVP 验证指标

| 指标 | 目标 |
|---|---|
| D1 次日留存 | ≥ 35% |
| D7 七日留存 | ≥ 15% |
| 内容记录率 | ≥ 60% |
| AI 互动率 | ≥ 40% |
| 用户完成一次完整归零流程比例 | ≥ 70% |

### 3.3 MVP 核心闭环

```text
打开 APP
  ↓
查看当前状态
  ↓
进行归零记录
  ↓
生成状态反馈
  ↓
ZEROON 陪伴回应
  ↓
沉淀到山海缓存
  ↓
再次打开
```

## 4. MVP 功能范围

MVP 只保留四个核心模块：

1. 首页
2. 归零
3. 山海缓存
4. 陪伴

明确不做：

- 社区
- 广场
- 点赞
- 评论
- 私信
- 电商
- 会员
- 广告
- 复杂任务管理
- 多人关系链

## 5. 模块一：首页

### 5.1 页面目标

首页用于展示用户当前状态，让用户快速进入归零流程。

首页不是数据看板，而是用户与 ZEROON 的每日第一次接触。

### 5.2 页面元素

| 元素 | 说明 |
|---|---|
| 当前状态 | 展示用户当前状态 |
| 发光核心 | 用颜色表达状态 |
| 今日寄语 | ZEROON 给用户的一句话 |
| 连续记录天数 | 鼓励用户持续使用 |
| 快速归零按钮 | 进入归零流程 |
| 今日山海缓存入口 | 查看最近沉淀内容 |

### 5.3 状态展示示例

```text
今天的 ZEROON：专注
发光核心：Soft Cyan
寄语：今天适合安静地完成一件重要的事。
连续归零：7 天
```

### 5.4 首页验收标准

- 用户打开 APP 后 3 秒内理解当前状态。
- 用户可以一键进入归零记录。
- 状态颜色与文案一致。
- 首页不展示复杂数据，不制造压力。

---

## 6. 模块二：归零

### 6.1 页面目标

帮助用户用最低成本完成一次状态记录。

### 6.2 输入内容

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| state | 枚举 | 是 | 用户当前状态 |
| goal | 文本 | 否 | 今日目标 |
| content | 文本 | 否 | 一句话记录 |

### 6.3 状态选项

- 平静
- 专注
- 创造
- 疲惫
- 混乱
- 高负荷

### 6.4 输出内容

归零完成后生成：

- 今日记录
- 状态变化
- AI 反馈
- 首页状态更新
- 山海缓存记录

### 6.5 归零流程

```text
选择状态
  ↓
输入一句话
  ↓
选择是否填写今日目标
  ↓
点击保存
  ↓
生成 ZEROON 反馈
  ↓
写入历史记录
```

### 6.6 验收标准

- 用户可在 30 秒内完成一次归零。
- 不强制用户写长文。
- 保存后立即更新首页状态。
- 记录必须可回看。

## 7. 模块三：山海缓存

### 7.1 页面目标

山海缓存用于保存用户的记录、状态变化和 AI 观察。

它不是社交内容流，而是用户自己的长期记忆库。

### 7.2 内容来源

- 用户归零记录
- AI 陪伴摘要
- 状态变化历史
- 月度回顾
- 年度回顾

### 7.3 展示方式

第一阶段采用时间轴：

```text
今天
昨天
本周
本月
更早
```

后续扩展：

- 月度回顾
- 年度回顾
- 关键词回看
- 状态趋势

### 7.4 内容类型

| 类型 | 说明 |
|---|---|
| Zero Record | 用户归零记录 |
| State Snapshot | 状态快照 |
| AI Reflection | AI 观察 |
| Monthly Summary | 月度总结 |

### 7.5 验收标准

- 用户可以查看历史归零记录。
- 用户可以按时间倒序查看内容。
- 用户可以看到状态变化。
- MVP 不支持公开发布。

---

## 8. 模块四：陪伴

### 8.1 页面目标

用户可以与 ZEROON 进行低压力对话。

陪伴模块不是问答机器人，而是长期状态陪伴入口。

### 8.2 AI 能力范围

MVP 支持：

- 情绪支持
- 反思引导
- 状态建议
- 目标追踪
- 简短陪伴回复

不支持：

- 医疗诊断
- 心理治疗
- 法律建议
- 金融建议
- 强任务型代理执行

### 8.3 对话原则

ZEROON 的回应必须：

- 安静
- 克制
- 温柔
- 不说教
- 不制造焦虑
- 不强行鸡汤
- 不过度拟人

### 8.4 示例

用户：

```text
今天很累，但还有很多事情没做。
```

ZEROON：

```text
先不用急着证明什么。
你可以先把今天归零。
剩下的事情，我们一件一件来。
```

### 8.5 验收标准

- 用户可以发送消息。
- ZEROON 可以根据当前状态回应。
- 对话记录可以保存。
- 首页状态可作为 AI 上下文。

## 9. 状态系统

### 9.1 状态定义

| 状态 | 颜色 | 含义 |
|---|---|---|
| 平静 | Warm Ivory | 恢复、休息、沉淀 |
| 专注 | Soft Cyan | 学习、工作、深度思考 |
| 创造 | Warm Gold | 灵感、创作、突破 |
| 疲惫 | Graphite Gray | 能量不足 |
| 高负荷 | Red | 压力过高 |
| 混乱 | Deep Night | 方向不清、需要整理 |

### 9.2 状态来源

状态来源包括：

1. 用户手动选择
2. 归零记录分析
3. AI 辅助判断
4. 未来硬件反馈

MVP 阶段以用户手动选择为主，AI 分析为辅。

### 9.3 状态变化规则

```text
用户记录状态
  ↓
生成本次状态
  ↓
与上一次状态对比
  ↓
写入 State History
  ↓
更新首页 Current State
```

### 9.4 验收标准

- 每次归零都生成状态记录。
- 当前状态可以快速读取。
- 历史状态可以追溯。
- 状态变化可用于统计和 AI 上下文。

## 10. 情绪灯联动

### 10.1 MVP 阶段

MVP 阶段不接入真实硬件。

APP 内只模拟发光核心状态。

### 10.2 V2 阶段

支持 BLE 情绪灯连接。

### 10.3 联动规则

| App 状态 | 情绪灯颜色 |
|---|---|
| 平静 | 白光 |
| 专注 | 蓝光 |
| 创造 | 金光 |
| 高负荷 | 红光 |
| 疲惫 | 灰白弱光 |
| 混乱 | 深蓝弱光 |

### 10.4 验收标准

MVP：

- 首页发光核心颜色随状态变化。

V2：

- APP 状态可同步到情绪灯。
- 情绪灯连接失败不影响 APP 主流程。

## 11. 用户系统

### 11.1 MVP 登录方式

优先支持：

- 手机号验证码登录

可选支持：

- 邮箱登录
- Apple 登录
- 微信登录

### 11.2 用户资料

| 字段 | 说明 |
|---|---|
| uid | 用户唯一 ID |
| nickname | 昵称 |
| avatar | 头像 |
| current_state | 当前状态 |
| continuous_days | 连续归零天数 |
| created_at | 注册时间 |

### 11.3 验收标准

- 用户可以注册登录。
- 用户资料可以读取。
- 用户状态和记录归属于用户。
- 用户退出后重新登录数据不丢失。

## 12. 数据与隐私

### 12.1 数据原则

用户记录默认私密。

MVP 不做公开内容。

### 12.2 数据范围

保存：

- 用户资料
- 归零记录
- 状态历史
- AI 对话记录
- 用户统计数据

不采集：

- 通讯录
- 精确定位
- 相册
- 麦克风
- 摄像头

除非用户明确授权并进入后续功能阶段。

### 12.3 验收标准

- 所有用户内容默认仅自己可见。
- API 必须鉴权。
- 用户只能访问自己的记录。
- 后台不可随意暴露用户私密内容。

---

## 13. 后台管理系统

### 13.1 MVP 后台范围

后台管理系统第一阶段只做基础功能：

- 用户列表
- 内容管理
- Prompt 管理
- 状态统计
- 系统配置

### 13.2 不做内容审核流

MVP 不开放社区，所以不需要复杂内容审核系统。

### 13.3 验收标准

- 管理员可以查看用户数量。
- 管理员可以配置官方内容。
- 管理员可以维护 AI Prompt。
- 后台必须有独立权限控制。

---

## 14. 运营内容与产品关系

### 14.1 运营栏目

ZEROON 运营侧已规划三个栏目：

- ZEROON 观察记录
- 山海缓存碎片
- 归零时刻

产品内的「山海缓存」可以承接这些内容栏目，但 MVP 阶段优先承接用户自己的记录。

### 14.2 内容进入产品的方式

第一阶段：

- 官方内容在后台录入
- App 展示精选内容

第二阶段：

- 用户记录生成个人山海缓存
- AI 生成周期性回顾

### 14.3 验收标准

- 官方内容与用户私密记录分离。
- 内容模块不影响归零主流程。

## 15. Roadmap

### Phase 1：MVP App

目标：

验证用户是否愿意持续记录和回看状态。

核心功能：

- 登录
- 首页状态
- 归零记录
- 山海缓存
- ZEROON 陪伴

### Phase 2：Emotion Light

目标：

让状态实体化。

核心功能：

- BLE 连接
- 情绪灯同步
- 状态颜色映射

### Phase 3：AI Voice Companion

目标：

让 ZEROON 拥有声音。

核心功能：

- 语音输入
- 语音回复
- 陪伴式语音交互

### Phase 4：Companion Plush

目标：

让 ZEROON 从数字产品进入实体陪伴。

核心功能：

- 发光核心
- NFC
- 蓝牙
- 玩偶交互

### Phase 5：ZEROON Ecosystem

目标：

形成完整陪伴生态。

## 16. MVP 验收总标准

MVP 完成标准：

- 用户可注册登录
- 用户可完成一次归零
- 用户可查看当前状态
- 用户可查看历史记录
- 用户可与 ZEROON 对话
- 状态可驱动发光核心表现
- 后台可配置基础内容
- 所有用户数据默认私密
- 核心流程无阻塞错误

完成以上标准即视为 ZEROON V1 MVP 可进入 Beta 测试。
