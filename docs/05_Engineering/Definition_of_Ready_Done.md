# Definition of Ready and Done

## Definition of Ready

A task can enter a sprint when:

- user value and scope are explicit;
- acceptance criteria include success, empty, loading, and error behavior;
- API and data changes are reviewed;
- privacy and authorization impact is identified;
- dependencies and owner are assigned;
- the task is estimated at no more than two engineer-days, or split further.

## Definition of Done

A task is done when:

- implementation and migration are committed;
- unit and relevant integration tests pass;
- API documentation is updated;
- authorization and user ownership are tested;
- logs contain no secrets or private message content;
- code review and CI pass;
- acceptance criteria are demonstrated in a runnable environment.
